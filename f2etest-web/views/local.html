为什么要代理到本地？

F2etest标准的hosts绑定方式是在hosts中进行编辑，但是代理到本地后，将可以获得以下额外优点：

  1. 复用本地的hosts绑定，一次绑定，F2etest中所有浏览器同时生效，保持两边一致，提高开发测试效率
  2. 解决办公网本机IP地址经常变动的问题，仅需将此组件设置为开机自动启动即可
  3. 可以在本地灵活的切换DNS，以方便切换公共环境
  4. 本地也可以绑定此代理，可彻底解决浏览器DNS缓存不生效问题，任何本地的hosts变更，都可以即时生效

如何代理到本地？

  1. 安装NodeJs：https://nodejs.org/
  2. 安装组件：npm install f2etest-local -g
  3. 开启代理：f2etest-local start --port 1080 --server http://<%=host%>/ --name <%=userid%> --apikey <%=apiKey%>
  4. 关闭代理：f2etest-local stop

f2etest-local详细介绍：https://www.npmjs.com/package/f2etest-local
